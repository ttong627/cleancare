# 크린케어(Clean Care) 통합 스마트 관리 시스템 V2.0 설계서

본 설계서는 현장 작업의 디지털 증빙과 사무 행정 자동화를 통합하여 크린케어의 업무 능률을 극대화하기 위한 시스템 구축 지침을 담고 있습니다. 
**V1.0 대비 보안/인증, 예외 처리, 데이터 동기화 안정성, 로깅 시스템이 대폭 강화되었습니다.**

## 1. 시스템 아키텍처 (System Architecture)
최상의 성능과 무지연(Zero-Loading), 안정성을 위해 클라우드 네이티브 환경과 하이브리드 부하 분산 구조를 채택합니다.

| 구분 | 기술 스택 / 아키텍처 | 핵심 역할 |
| --- | --- | --- |
| **Cloud Infra** | Google Cloud Platform (Cloud Run, Firestore, Storage, **Cloud Logging**) | 실시간 동기화, 오토 스케일링, 무중단 배포, **에러/크래시 중앙 집중 모니터링** |
| **Mobile App** | Flutter (Dart), Isar Database (Local), **Firebase Crashlytics** | AR 도면 생성, 오프라인 퍼스트 촬영, 모바일 팩스, **실시간 앱 크래시 추적** |
| **PC Web** | React (Next.js), Tailwind CSS, **React Query / Zustand** | 예외 중심 통합 관제, 행정/회계 업무 자동화, **데이터 캐싱 및 무지연 UI 렌더링** |
| **External API** | 카카오내비, OpenWeatherMap, 국세청 API | 길 안내, 날씨 정보, 세금계산서 연동 |
| **Auth & Security**| **Firebase Authentication, Firestore Security Rules** | **관리자/작업자 역할 기반 접근 제어(RBAC), 데이터 위변조 방지** |

## 2. 핵심 기능 상세 설계 (Core Features)

### 2.1 AR 기반 현장 관리 (Mobile)
*   **동영상 도면 스캔:** LiDAR 및 Depth API를 활용하여 카메라 스캔만으로 실시간 2D 평면도 생성.
*   **AR 촬영 마커:** 공간 내 촬영 위치(핀)를 직접 지정하며, 학교(전/중/후) 및 LH(전/후) 모드별 촬영 강제.
*   **AI 사진 검증:** '전' 사진의 데이터를 오버레이(Ghosting)하여 '후' 사진 촬영 시 픽셀 단위 구도 일치 확인(15% 오차 시 재촬영 유도).
*   **오프라인 작업 및 충돌 방지:** 통신 음영 지역에서 로컬 DB 선저장 후 네트워크 연결 시 자동 동기화. **서버 타임스탬프(Server Timestamp)를 활용하여 다중 기기 접속 시 데이터 충돌(Conflict) 완벽 해결.**

### 2.2 통합 관제 및 행정 자동화 (PC)
*   **예외 중심(Exception-First) 대시보드:** 지각, AI 반려 등 즉각 조치가 필요한 이슈 현장을 최상단에 배치하여 관리 집중도 향상. **웹소켓(Firestore Listener) 기반 무지연 실시간 알림 적용 및 좀비 데이터(자동 시드 복구) 완벽 차단.**
*   **캘린더 기반 작업 등록 (Firebase 완전 동기화):** 일자별 캘린더에서 현장을 원클릭으로 DB에 직접 등록 및 영구 삭제. **React Local State가 아닌 상용 클라우드 DB 연동으로 새로고침 시에도 완벽한 데이터 영속성 보장.**
*   **카카오맵 실시간 위치 관제:** 자동 주소-좌표 변환(Geocoding)을 통해 지도 위에 작업 현장 및 현재 투입된 실무자의 위치를 실시간 클러스터링으로 표시.
*   **원스톱 보고서 생성:** 현장 사진과 도면 정보를 학교/LH 전용 정산 템플릿에 자동 맵핑하여 PDF 생성.
*   **원클릭 행정 처리:** 작업서 전송(팩스/메일), 견적서 발행, 국세청 세금계산서 전송을 단일 워크플로우로 통합.

### 2.3 시스템 보안 및 데이터 무결성 관리
*   **원클릭 DB 백업 & 복원:** 관리자 설정 메뉴에서 전체 Firestore DB 데이터를 `.json` 파일 포맷으로 로컬에 다운로드하고 복원할 수 있는 기능 탑재 (랜섬웨어 및 데이터 유실 원천 차단).
*   **강력한 역할 기반 접근 제어 (RBAC):** 작업자 전용 모드 강제로 데이터 위변조 방지.

### 2.4 모바일/PC 통합 팩스
*   **스마트 송신:** 휴대폰 내 문서(PDF, IMG) 선택 및 수신처 번호 입력 후 즉시 송신.
*   **상태 동기화:** 송신 성공/실패 여부를 푸시 알림으로 수신하고 PC와 이력을 통합 공유.

## 3. 상세 화면(UI/UX) 구성

### 3.1 모바일 작업자 화면
*   **홈 화면:** 실시간 날씨 기반 작업 추천, 카카오내비 연동 '길 안내' 버튼, 오늘의 작업 리스트.
*   **촬영 화면:** 전체 AR 뷰어, 우측 상단 2D 미니맵, 하단 단계별(전/중/후) 촬영 탭 및 셔터.

### 3.2 PC 관리자 화면
*   **관제 메인:** 적색(긴급 조치), 청색(진행 중), 녹색(완료) 색상 코드로 현장 상태 시각화. **다크모드/라이트모드 기본 지원으로 장시간 관제 피로도 최소화.**
*   **정산 모달:** 사진 검수 완료 후 견적 금액 확인 및 세금계산서/팩스/메일 일괄 발송 버튼 배치.
*   **거래처 및 API 관리:** 다중 연락처 관리 시스템 및 API 키(팩스/국세청 등) 안전한 로컬/클라우드 영구 저장 기능.

### 3.3 브랜드 아이덴티티 및 디자인 시스템 (Design System)
*   **Stitch 디자인 시스템 (Pure Blue Clean):** 신뢰감과 청결함을 상징하는 Deep Ocean Blue & Sky Blue 톤을 모바일 앱과 PC 웹 전체 UI에 일괄 적용 (Tailwind v4 유틸리티 연동).
*   **Anti-Gravity 랜딩 페이지:** 모던하고 인터랙티브한 애니메이션이 적용된 기업 소개/대시보드 진입점 구축.
*   **통합 로고 최적화:** 모든 디바이스에서 가독성이 보장되도록 최적화된 기업 통합 로고 자산 적용.

## 4. 데이터베이스 및 성능 설계 (Performance & Security)

### 4.1 고도화된 Firestore 데이터 스키마 (보안 및 이력 관리 강화)
*   프로젝트 정보에 **생성/수정 시간(createdAt, updatedAt)** 및 **작업자 ID(workerId)**, **반려 사유(rejectReason)** 추가.
*   상세 스키마 구조는 첨부된 `schema.ts` 참고.

### 4.2 안정성 보장 아키텍처
*   **강력한 접근 제어:** Firestore Security Rules를 통해 관리자는 모든 권한을, 작업자는 본인에게 할당된 프로젝트 데이터만 수정 가능하도록 제한.
*   **부하 분산:** 무거운 AI 연산은 Cloud Run 서버에서 비동기로 처리하여 모바일 발열 방지.
*   **데이터 수명 관리:** 완료 후 1개월 경과 데이터는 Cloud Storage 아카이브로 자동 이관하여 대시보드 로딩 속도 최적화.
*   **이미지 최적화:** 촬영 직후 WebP 포맷 압축 및 리사이징을 거쳐 서버로 전송 (트래픽 80% 절감, Zero-Loading 구현).
